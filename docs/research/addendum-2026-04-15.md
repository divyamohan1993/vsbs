# Research Addendum — delta from parallel specialist agents (2026-04-15)

Five specialist research agents ran in parallel and delivered their reports after the initial research pass. Their deltas are folded in here and into the cited specialist docs. Citations are the agents'; every URL is live as of 2026-04-15.

## A. Agentic — LangGraph elevated; BFCL V4 numbers; A-MEM; GraphRAG cost

1. **LangGraph is the most production-mature orchestrator** as of April 2026 — first-class checkpointing, crash recovery, time-travel debugging. Deployed at Uber, Klarna, LinkedIn, JPMorgan and 400+ others ([gurusup 2026](https://gurusup.com/blog/best-multi-agent-frameworks-2026), [LangChain docs](https://docs.langchain.com/oss/python/deepagents/comparison), [Morph LLM framework review](https://www.morphllm.com/ai-agent-framework)).
2. **Claude Agent SDK** remains best-in-class for sub-agents needing deep OS/tool/shell access (diagnostics parsing, document handling).
3. **Decision update:** use LangGraph as the outer orchestrator (hierarchical supervisor + pipeline segments + Postgres checkpoints) and embed Claude Agent SDK inside the DiagnosisAgent and FulfillmentAgent sub-agents where OS-level tool depth matters. Claude Managed Agents beta remains the hosted fallback for teams that do not want to own the runtime.
4. **BFCL V4 (2026-04-12 snapshot):** Claude Opus 4.1 #2 @ 70.36 %, Sonnet 4 #3 @ 70.29 %, GPT-5 #7 @ 59.22 % ([BFCL leaderboard](https://gorilla.cs.berkeley.edu/leaderboard.html), [llm-stats](https://llm-stats.com/benchmarks/bfcl)). τ2-bench: GPT-5.4 at 98.9 for tool-heavy desktop automation ([mindstudio comparison](https://www.mindstudio.ai/blog/gpt-54-vs-claude-opus-46-vs-gemini-31-pro-benchmarks)). Gemini 3.1 Pro leads BrowseComp at 86 ([benchlm](https://benchlm.ai/blog/posts/gpt5-vs-gemini-2026)).
5. **Memory:** Mem0 (arXiv 2504.19413, ECAI 2025) confirmed as the production winner over LangMem (p95 ~60 s search latency, impractical for real-time advisor) and A-MEM (arXiv 2502.12110, NeurIPS 2025) which is academic SOTA but operationally heavier.
6. **GraphRAG cost:** Microsoft's reference implementation costs ~$33 K to index large corpora; 2025–26 optimised variants cut that 90 % ([Graph Praxis cost cut](https://medium.com/graph-praxis/cutting-graphrag-token-costs-by-90-in-production-5885b3ffaef0), [Graph Praxis 2026 survey](https://medium.com/graph-praxis/graph-rag-in-2026-a-practitioners-guide-to-what-actually-works-dca4962e7517)).
7. **Routing:** unified routing+cascade + speculative cascades ([arXiv 2410.10347](https://arxiv.org/abs/2410.10347), [Google Research speculative cascades](https://research.google/blog/speculative-cascades-a-hybrid-approach-for-smarter-faster-llm-inference/)) — validated numbers: up to 40 % fewer large-model calls at equal quality. Matches our Haiku-first-then-Opus design.

## B. Dispatch & GCP — two important factual corrections

1. **Google Maps Directions API and Distance Matrix API are LEGACY** as of **1 March 2025**. New projects cannot enable them. Both are superseded by **Routes API v2** (`:computeRoutes` and `:computeRouteMatrix`) ([Routes API](https://developers.google.com/maps/documentation/routes), [Maps deprecations](https://developers.google.com/maps/deprecations), [March 2025 billing changes](https://developers.google.com/maps/billing-and-pricing/march-2025), [migration guide](https://developers.google.com/maps/documentation/routes/migrate-routes)). **Our adapter at `apps/api/src/adapters/maps.ts` already calls `:computeRoutes` and was correct by construction**; the `dispatch.md` reference to "Distance Matrix" is updated to **computeRouteMatrix on Routes API v2**.
2. **Google Maps ETA (TRAFFIC_AWARE_OPTIMAL) is GNN-backed.** DeepMind + Google Maps collaboration improved ETA accuracy by up to **50 %** in major cities using graph neural networks over "Supersegments" ([DeepMind blog](https://deepmind.google/discover/blog/traffic-prediction-with-advanced-graph-neural-networks/), [arXiv 2108.11482](https://arxiv.org/abs/2108.11482)). We do not need to train our own ETA model — use `TRAFFIC_AWARE_OPTIMAL` on Routes API.
3. **Route Optimization API (GMPRO)** is GA as expected, wrapping Google's VRP solver with hard+soft time windows, capacity, breaks, workload balancing ([GMPRO overview](https://developers.google.com/maps/documentation/route-optimization/overview), [GA blog](https://mapsplatform.google.com/resources/blog/plan-efficient-routes-for-your-fleet-route-optimization-api-is-now-generally/), [time windows](https://developers.google.com/maps/documentation/route-optimization/concepts/time-windows)).
4. **Cloud KMS has launched quantum-safe KEM and quantum-safe digital signatures** ([KEM announcement](https://cloud.google.com/blog/products/identity-security/announcing-quantum-safe-key-encapsulation-mechanisms-in-cloud-kms), [signatures announcement](https://cloud.google.com/blog/products/identity-security/announcing-quantum-safe-digital-signatures-in-cloud-kms)). **ML-KEM-768, ML-DSA-65, SLH-DSA** are available on GCP today. Our `security.md` claim that PQ is deployed at Cloud KMS is now confirmed and not "when GA" — it is GA. Update accordingly.
5. **Vertex AI Agent Builder + Agent Engine are GA** for production agents ([Agent Builder](https://cloud.google.com/products/agent-builder), [release notes](https://docs.cloud.google.com/vertex-ai/generative-ai/docs/release-notes)).
6. **Gemini version drift:** **Gemini 2.5 Pro/Flash/Flash-Lite retire 16 Oct 2026.** **Gemini 3 Pro/Flash is the current production model.** Update `.env.example` and architecture to default to `gemini-3-pro`.
7. **Stochastic / dynamic VRPTW** survey 2025 confirms our formulation is standard ([MDPI Nov 2025](https://www.mdpi.com/2227-7390/13/23/3782), [MDPI 2024 urban review](https://www.mdpi.com/2227-7390/12/1/28), [Pavone et al.](https://stanfordasl.github.io/wp-content/papercite-data/pdf/Pavone.Bisnik.ea.MONE09.pdf)).
8. **RL for appointment allocation:** GTARS-DRL ([Springer 2024](https://link.springer.com/article/10.1007/s10791-024-09474-1)), CPS ([Nature Sci Rep 2024](https://www.nature.com/articles/s41598-024-77873-x)). Useful for a future v1.1 capacity-learning component.
9. **Mobile-mechanic reality check for India:** GoMechanic's model is **pickup + drop**, not home mobile mechanics, because spare-parts logistics dominate labour cost ([GoMechanic](https://gomechanic.in/)). **Architect dispatch as three modes (mobile / pickup-drop / tow), not two.** Our `DispatchDecisionSchema` already includes `pickup-drop` as a first-class mode via `PickupModeSchema` — confirmed correct.
10. **Wrench + YourMechanic merged 2022** and dispatch is **ZIP-whitelist based**, not a circular radius ([PR](https://www.prnewswire.com/news-releases/wrench-acquires-mobile-vehicle-repair-network-yourmechanic-301560120.html)). Update our amber-mobile logic to optionally plug a ZIP/pincode allow-list.

## C. Automotive — India reality, open DTC corpus, BS6 Phase 2

1. **BS6 Phase 2 (1 Apr 2023)** mandates OBD-II on **two-wheelers and heavy commercial vehicles** in India; all post-2023 four-wheelers are already OBD-II compliant ([TVS explainer](https://www.tvsmotor.com/media/blog/bs6-phase-2-rde-and-obd-2-compliance-explained), [BikeDekho OBD-2B](https://www.bikedekho.com/news/everything-you-need-to-know-about-obd-2b-compliance-17594)).
2. **Smartcar / High Mobility / Otonomo do NOT cover India.** Production path for India is **own BLE ELM327 / vLinker MS dongle gateway**, plus OEM telematics where available (Maruti-MapMyIndia, Hyundai BlueLink, Tata iRA, Mahindra AdrenoX) — no unified API across them. Update `automotive.md` to reflect this hard constraint.
3. **India RC lookup aggregators** confirmed: **Surepass Vahan Verification API**, **Signzy Vehicle RC API**, **Karza / Perfios** — pricing tiered per-call, contract-based. Our adapter interface already accommodates this; recommend the operator pick one at deployment.
4. **NHTSA TSB dataset only provides summaries** — full TSB PDFs remain OEM-copyrighted. Weekly bulk ZIP at `data.transportation.gov/Automobiles/.../hczg-qbhf`. Our RAG corpus ingestor must respect this (summaries public, bodies licensed).
5. **India recalls:** SIAM voluntary recall portal + VAHAN vehicle-recall page are **HTML-only, no API**. Ingestion is a scheduled scrape that respects robots.txt.
6. **Open DTC database:** `github.com/Wal33D/dtc-database` ships **28,220 codes (9,415 generic + 18,805 mfr-specific, 33 brands)** as an MIT-licensed SQLite — we can ship this directly in `packages/kb` as the shipped generic DTC resolver. `github.com/todrobbins/dtcdb` and `github.com/mytrile/obd-trouble-codes` are additional open mirrors.
7. **Symptom → diagnosis ML literature (2024–2025)** to cite: Khoshkangini et al., *Automotive fault nowcasting with ML and NLP*, Springer *Machine Learning* 2024 ([DOI](https://link.springer.com/article/10.1007/s10994-023-06398-7)); *Deep Learning-Based Vehicle DTC Prognosis*, Int. ## D. Wellbeing — additional peer-reviewed anchors

1. **Driver stress biometrics:** Mehler, Reimer & Coughlin 2012, *Human Factors*, [DOI 10.1177/0018720812442086](https://doi.org/10.1177/0018720812442086). NHTSA Driver Workload Metrics Project DOT HS 810 635 (2006). SAE J2944. Driver Behaviour Questionnaire — Reason, Manstead, Stradling, Baxter & Campbell 1990, *Ergonomics*, [DOI 10.1080/00140139008925335](https://doi.org/10.1080/00140139008925335). Use DBQ as the instrument for our self-safety questionnaire.
2. **Wait uncertainty > wait length:** Taylor 1994 *Journal of Marketing*, [DOI 10.2307/1252269](https://doi.org/10.2307/1252269); Pruyn & Smidts 1998 *IJRM*, [DOI 10.1016/S0167-8116(98)00008-1](https://doi.org/10.1016/S0167-8116(98)00008-1). Strongly supports our "accuracy beats speed" UX law.
3. **SERVQUAL automotive specialisation:** Bouman & van der Wiele 1992 *IJSIM*, [DOI 10.1108/09564239210015157](https://doi.org/10.1108/09564239210015157) — reliability and empathy as dominant loyalty predictors. Complements Izogo & Ogba 2015.
4. **Cost Transparency Index — supporting literature:** Sinha 2000 *HBR*; Miller 2019 *JCR*, [DOI 10.1093/jcr/ucz020](https://doi.org/10.1093/jcr/ucz020); Carter & Curry 2010. Our CTI formula is compatible with both Carter/Curry and Miller.
5. **WCAG 2.2 correction:** The AAA enhanced focus-appearance criterion is **2.4.13 Focus Appearance (AAA)**, not 2.4.12 — which is 2.4.12 Focus Not Obscured (Enhanced, AAA, new in 2.2). **2.4.11 Focus Not Obscured (Minimum) is AA.** Also enforce **3.3.8 Accessible Authentication (Enhanced, AAA)** — no cognitive function test at all. Update `frontend.md` / `wellbeing.md` accordingly.
6. **49 CFR §577** is the US regulatory anchor for do-not-drive advisories on imminent-hazard defects. Cite alongside our red-flag override logic in `wellbeing.md` §4.
7. **Trust in AI advisor 2024–2026:** Glikson & Woolley 2020 *AoM Annals*, [DOI 10.5465/annals.2018.0057](https://doi.org/10.5465/annals.2018.0057) (already cited); **Bach, Khan, Hallock, Beltrão & Sousa 2024** "A systematic literature review of trust in AI," *AI & Society*, [DOI 10.1007/s00146-022-01552-6](https://doi.org/10.1007/s00146-022-01552-6); **Kim, Lee & Park 2024** CHI '24 on explainability-trust link, [DOI 10.1145/3613904.3642474](https://doi.org/10.1145/3613904.3642474); **Shin 2022** *IJHCS*, [DOI 10.1016/j.ijhcs.2020.102551](https://doi.org/10.1016/j.ijhcs.2020.102551) — fairness + transparency beat accuracy for trust. These replace any weaker trust citations.
8. **NEW: fairness incompatibility.** Chouldechova 2017 *Big Data*, [DOI 10.1089/big.2016.0047](https://doi.org/10.1089/big.2016.0047) — formal proof that demographic parity, equalised odds, and calibration are pairwise incompatible when base rates differ across groups. Implication: our fairness gate must **pick one metric per axis and commit to it**, not silently optimise several. We commit to **equalised odds** on dispatch quality across geography and vehicle-age bands, with demographic-parity as a secondary monitor.

## E. Frontend — Yjs, Gemini Live API, realtime on GCP

1. **Yjs + y-indexeddb** is the better CRDT choice for our domain than Automerge — binary encoding, GC'd, handles our structured booking state faster while Automerge's JSON ergonomics don't buy us anything for tightly-typed data ([Yjs](https://github.com/yjs/yjs), [velt CRDT comparison](https://velt.dev/blog/best-crdt-libraries-real-time-data-sync), [Convex comparison](https://stack.convex.dev/automerge-and-convex)). **Update `frontend.md` and `STACK.md`: Yjs instead of Automerge 3.**
2. **Dexie** remains for non-CRDT caches (vehicle catalogue, photo blobs) — used by WhatsApp Web, MS To Do, GitHub Desktop ([dexie.org](https://dexie.org/)).
3. **Realtime on Cloud Run** — Cloud Run supports WebSockets but caps 1000 concurrent per container with scaling quirks ([Cloud Run WS docs](https://docs.cloud.google.com/run/docs/triggering/websockets)). **Ably** is the hosted option that gives WS + SSE + MQTT + HTTP streaming, exactly-once, history, rewind, presence, global edge ([Ably vs Cloud Run](https://ably.com/compare/google-cloud-run-vs-pusher), [index.dev compare](https://www.index.dev/skill-vs-skill/pusher-vs-ably-vs-pubnub)). **Decision update:** SSE stays primary on our own Cloud Run for the status ticker (simple, cheap, proxy-friendly on Indian ISPs); Ably is the escalation if we need presence/history/rewind. The autonomy dashboard's bidirectional camera/sensor stream goes through a dedicated Ably channel where the operator enables it.
4. **Gemini 2.5 Flash Native Audio Live API** (native audio in/out, transcripts as side-channel) is GA on Vertex AI — use it for conversational voice intake without an STT/TTS hop ([Google blog](https://blog.google/products/gemini/gemini-audio-model-updates/), [ai.google.dev Live API](https://ai.google.dev/gemini-api/docs/live-api), [Vertex GA blog](https://cloud.google.com/blog/products/ai-machine-learning/gemini-live-api-available-on-vertex-ai)). **Preview model `gemini-live-2.5-flash-native-audio` was deprecated 2026-03-19** — migrate to the GA live model ([Vertex Live docs](https://docs.cloud.google.com/vertex-ai/generative-ai/docs/models/gemini/2-5-flash-live-api)). `.env.example` gets a `GEMINI_LIVE_MODEL` key.
5. **Radix UI was acquired by WorkOS**, updates on complex components have slowed ([pkgpulse](https://www.pkgpulse.com/blog/shadcn-ui-vs-base-ui-vs-radix-components-2026)). Primitives remain stable; we stay on Radix for now.
6. **shadcn/ui Charts have an accessibility audit gap** ([ashleemboyer audit](https://ashleemboyer.com/blog/a-quick-ish-accessibility-review-shadcn-ui-charts/)) — we replace shadcn Charts with Recharts + manual ARIA for any chart in the admin dashboard.
7. **React 19.2** (released 2025-10-01) lands INP optimisation + Activity API for off-screen work ([React 19.2 blog](https://react.dev/blog/2025/10/01/react-19-2)). Our `<Activity>` usage should apply to the autonomy-status dashboard so off-screen tabs stop rendering without losing state.
