# Support

VSBS is a research-grade project under active development. Here is where to take your question so it lands in the right place and gets an answer.

## Quick triage
 
| Your situation | Go here |
|---|---|
| "How do I run it?" | [`README.md`](README.md) → Quick start |
| "How does X work?" | [`docs/`](docs/) — start with `architecture.md` then drill into the relevant `research/*.md` |
| "Is X real or simulated today?" | [`docs/gap-audit.md`](docs/gap-audit.md) |
| "What should I build next?" | [`docs/roadmap-prod-deploy.md`](docs/roadmap-prod-deploy.md) |
| "Is this feature planned?" | [GitHub Discussions → Ideas](https://github.com/divyamohan1993/vsbs/discussions/categories/ideas) |
| "I found a bug" | [New issue → Bug report](https://github.com/divyamohan1993/vsbs/issues/new?template=bug_report.yml) |
| "I want a new feature" | [New issue → Feature request](https://github.com/divyamohan1993/vsbs/issues/new?template=feature_request.yml) |
| "I need help running or extending VSBS" | [GitHub Discussions → Q&A](https://github.com/divyamohan1993/vsbs/discussions/categories/q-a) |
| "I want to discuss the research" | [GitHub Discussions → Research](https://github.com/divyamohan1993/vsbs/discussions/categories/research) |
| "I found a security issue" | [Private advisory](https://github.com/divyamohan1993/vsbs/security/advisories/new) — never a public issue |
| "I represent an OEM or want a partnership" | `contact@dmj.one` |

## What we can help with

- Running the repo locally (`sim` profile, zero API keys).
- Interpreting the research docs, compliance pack, and stack choices.
- Extending adapters (payments, SMS, maps, OEM telematics, LLM providers).
- DPDP / GDPR / EU AI Act posture questions at the conceptual level.
- Integration architecture reviews for partners who want to adopt VSBS.

## What we cannot help with

- Legal advice. The compliance pack is a starting point, not a substitute for a DPO or counsel.
- Jurisdiction-specific certification paperwork. That is your regulator's domain.
- Debugging your own fork's divergence from `main`. Upstream the change or open a discussion.
- Generic "how does React / Bun / Hono work" questions. Please ask those upstream.

## Response expectations

This is a solo-maintained project today. Best-effort response windows:

- **Security advisories:** within 72 hours.
- **Issues and discussions:** within 7 days.
- **Pull requests:** within 14 days to first review. Larger or research-grounded PRs may take longer.

## Paid support / partnership

For OEM pilots, safety reviews, custom sensor-fusion / PHM plug-ins, regional compliance engagements, or research collaborations, email `contact@dmj.one`. See [`NOTICE`](NOTICE) for the kind of partnerships the project is open to.
